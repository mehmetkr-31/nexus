`streamingPlugin()` adds a WebSocket-based streaming hook to your `nexus` client. Unlike `useContracts` (which polls via HTTP), `useStreamContracts` maintains a live connection and applies creates and archives in real time — zero latency, no polling interval.

This is ideal for order books, trading dashboards, collaborative apps, or any UI where users need to see contract state change as it happens.

## Installation

<Steps>
  <Step>

  ### Install the plugin

  `streamingPlugin` ships in `@nexus-framework/react` — no extra package needed:

  ```ts title="lib/nexus-client.ts"
  "use client";

  import {
    createNexusClient,
    sandboxAuth,
    tanstackQueryPlugin,
    streamingPlugin, // [!code highlight]
  } from '@nexus-framework/react';

  export const nexus = createNexusClient({
    baseUrl: process.env.NEXT_PUBLIC_CANTON_API_URL ?? 'http://localhost:7575',
    plugins: [
      sandboxAuth({
        secret: process.env.NEXT_PUBLIC_SANDBOX_SECRET ?? 'secret',
        userId: process.env.NEXT_PUBLIC_SANDBOX_USER_ID ?? 'alice',
        partyId: process.env.NEXT_PUBLIC_SANDBOX_PARTY_ID ?? 'Alice::...',
      }),
      tanstackQueryPlugin(),   // keep — needed for useContracts etc.
      streamingPlugin(),       // adds nexus.useStreamContracts // [!code highlight]
    ],
  });
  ```

  </Step>
  <Step>

  ### Add NexusProvider

  `nexus.NexusProvider` must wrap any component tree that uses `nexus.*` hooks:

  ```tsx title="app/layout.tsx"
  import { nexus } from '@/lib/nexus-client';

  export default function RootLayout({ children }: { children: React.ReactNode }) {
    return (
      <html>
        <body>
          <nexus.NexusProvider>
            {children}
          </nexus.NexusProvider>
        </body>
      </html>
    );
  }
  ```

  </Step>
</Steps>

## Usage

### Basic usage

```tsx title="components/LiveIouList.tsx"
"use client";

import { nexus } from '@/lib/nexus-client';

interface IouPayload {
  owner: string;
  amount: string;
  currency: string;
}

// Pass a stable reference — module constant avoids reconnects on every render
const PARTIES = ['Alice::122059a10c...'];

export function LiveIouList() {
  const { contracts, isLive, error } = nexus.useStreamContracts<IouPayload>({
    templateId: 'my-package:Iou:Iou',
    parties: PARTIES,
  });

  if (error) {
    return <p style={{ color: 'red' }}>Stream error: {error.message}</p>;
  }

  return (
    <div>
      <h2>
        Live Iou contracts ({contracts.length})
        {isLive ? (
          <span style={{ color: '#16a34a', fontSize: 12, marginLeft: 8 }}>● Live</span>
        ) : (
          <span style={{ color: '#9ca3af', fontSize: 12, marginLeft: 8 }}>Connecting…</span>
        )}
      </h2>
      <ul>
        {contracts.map((c) => (
          <li key={c.contractId}>
            {c.payload.amount} {c.payload.currency}
            <code style={{ marginLeft: 8, fontSize: 11 }}>
              {c.contractId.slice(0, 12)}…
            </code>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### Conditional streaming

Don't open a WebSocket until the user is authenticated:

```tsx
"use client";

import { useSession } from '@/hooks/use-session';
import { nexus } from '@/lib/nexus-client';
import { useMemo } from 'react';

export function AuthenticatedLiveList() {
  const { session, isLoading: sessionLoading } = useSession();

  // Memoize parties so the array reference stays stable
  const parties = useMemo(
    () => (session ? [session.partyId] : undefined),
    [session?.partyId],
  );

  const { contracts, isLive, error } = nexus.useStreamContracts({
    templateId: 'my-package:Iou:Iou',
    parties,
    enabled: !!session && !sessionLoading, // don't stream until authenticated
    onLive: () => console.log('ACS snapshot complete, now live'),
    onError: (err) => console.error('Stream error:', err),
  });

  if (!session) return <p>Please log in</p>;

  return (
    <div>
      {!isLive && <p>Loading initial contract set…</p>}
      {isLive && <p>{contracts.length} active contracts</p>}
    </div>
  );
}
```

### Manually closing the stream

The stream closes automatically when the component unmounts. You can also close it manually — for example, after the user navigates away or pauses streaming:

```tsx
"use client";

import { useState } from 'react';
import { nexus } from '@/lib/nexus-client';

export function PausableStream() {
  const [paused, setPaused] = useState(false);

  const { contracts, isLive, close } = nexus.useStreamContracts({
    templateId: 'my-package:Iou:Iou',
    enabled: !paused,
  });

  return (
    <div>
      <button onClick={() => {
        if (!paused) close(); // close the WebSocket
        setPaused((p) => !p);
      }}>
        {paused ? 'Resume' : 'Pause'}
      </button>
      <p>{isLive ? `${contracts.length} contracts (live)` : 'Stream closed'}</p>
    </div>
  );
}
```

## Return value

```ts
interface StreamContractsState<T> {
  /**
   * The current live set of active contracts.
   * Creates are added, archives are removed in real time.
   */
  contracts: ActiveContract<T>[];

  /**
   * True once the initial ACS snapshot has been fully delivered.
   * Before this point, `contracts` is partially populated.
   */
  isLive: boolean;

  /**
   * Set if the WebSocket encountered an error.
   * The stream is NOT automatically retried — re-mount or toggle `enabled`.
   */
  error: Error | null;

  /**
   * Call to manually close the WebSocket.
   * The stream closes automatically on component unmount.
   */
  close: () => void;
}
```

## How it works

<Steps>
  <Step>

  ### Connection & auth

  When `enabled` is `true`, the hook calls `CantonClient.streamActiveContracts()`, which opens a WebSocket to:

  ```
  ws://your-ledger/v2/state/active-contracts/stream
  ```

  Authentication uses the **`jwt.token.<TOKEN>`** WebSocket subprotocol — the Canton standard for token-based WebSocket auth:

  ```
  Sec-WebSocket-Protocol: jwt.token.eyJ..., daml.ws.auth
  ```

  </Step>
  <Step>

  ### Subscription message

  After the connection opens, a subscription message is sent:

  ```json
  {
    "templateIds": ["my-package:Iou:Iou"],
    "parties": ["Alice::122059a10c..."]
  }
  ```

  </Step>
  <Step>

  ### ACS snapshot

  Canton immediately streams all currently active contracts matching the subscription as `created` messages. The hook accumulates these into the `contracts` array. Duplicates (by `contractId`) are skipped.

  </Step>
  <Step>

  ### Live marker

  Once the snapshot is complete, Canton sends a live marker. The hook sets `isLive = true` and calls `onLive()`.

  </Step>
  <Step>

  ### Real-time updates

  From this point, `created` events add to `contracts` and `archived` events remove from it. The component re-renders on every change.

  </Step>
  <Step>

  ### Cleanup

  On component unmount (or when `enabled` becomes `false`), the WebSocket is closed via `ws.close()`.

  </Step>
</Steps>

## useStreamContracts vs useContracts

| | `useContracts` | `useStreamContracts` |
|--|--|--|
| Transport | HTTP (REST) | WebSocket |
| Latency | `staleTime` + refetch | Sub-second |
| Cache | TanStack Query | Local React state |
| Offline recovery | Automatic retry | Manual — re-mount or toggle `enabled` |
| SSR prefetch | ✓ (via `prefetchContracts`) | ✗ (browser-only) |
| Ideal for | Standard CRUD, admin pages | Live dashboards, order books, collaboration |

<Callout>
  `streamingPlugin` and `tanstackQueryPlugin` are fully independent. `useStreamContracts` does **not** read from or write to the TanStack Query cache. You can use both hooks for the same template — one for prefetched server data, the other for live updates.
</Callout>
