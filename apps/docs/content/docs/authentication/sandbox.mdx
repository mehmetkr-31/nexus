The sandbox auth plugin generates signed JWT tokens accepted by Canton Sandbox's built-in authenticator. It's the fastest way to get started in development — no key infrastructure, no OAuth setup, no token endpoint to configure.

<Callout type="warn">
  Use `sandboxAuth` only in development. For production, use [`jwtAuth`](/docs/authentication/jwt) or [`oidcAuth`](/docs/authentication/oidc).
</Callout>

## Installation

<Steps>
  <Step>

  ### Add to your server client

  ```ts title="lib/nexus.ts"
  import { createNexus, sandboxAuth } from '@nexus-framework/core'; // [!code highlight]

  export const nexus = createNexus({
    ledgerApiUrl: 'http://localhost:7575',
    plugins: [
      sandboxAuth({ // [!code highlight]
        secret: 'secret',               // must match Canton's --auth-secret
        userId: 'alice',
        partyId: 'Alice::122059a10c...',
      }), // [!code highlight]
    ],
  });
  ```

  </Step>
  <Step>

  ### Add to your React client

  For Next.js and other React frameworks, use `createNexusClient` from `@nexus-framework/react`:

  ```ts title="lib/nexus-client.ts"
  "use client";

  import { createNexusClient, sandboxAuth, tanstackQueryPlugin } from '@nexus-framework/react'; // [!code highlight]

  export const nexus = createNexusClient({
    baseUrl: process.env.NEXT_PUBLIC_CANTON_API_URL ?? 'http://localhost:7575',
    plugins: [
      sandboxAuth({ // [!code highlight]
        secret: process.env.NEXT_PUBLIC_SANDBOX_SECRET ?? 'secret',
        userId: process.env.NEXT_PUBLIC_SANDBOX_USER_ID ?? 'alice',
        partyId: process.env.NEXT_PUBLIC_SANDBOX_PARTY_ID ?? 'Alice::...',
      }), // [!code highlight]
      tanstackQueryPlugin(),
    ],
  });
  ```

  </Step>
  <Step>

  ### Get your partyId

  Query the Canton Sandbox admin API to find your partyId:

  ```bash
  curl http://localhost:7575/v2/admin/parties | jq '.parties[] | select(.isLocal) | .partyId'
  ```

  Or start Canton Sandbox with a deterministic party:

  ```bash
  docker run -p 7575:7575 \
    -e CANTON_SANDBOX_USER=alice \
    digitalasset/canton-sandbox:latest
  ```

  </Step>
</Steps>

## Usage

Once configured, all ledger calls automatically attach a fresh signed token:

```ts
// Token is generated and attached automatically — no manual getToken() call needed
const { contracts } = await nexus.ledger.contracts.fetchActiveContracts({
  templateId: 'my-package:Iou:Iou',
});
```

## Options

<TypeTable
  type={{
    secret: {
      description: "HMAC256 secret — must match Canton's `--auth-secret` flag.",
      type: "string",
      required: true,
    },
    userId: {
      description: "Canton userId (e.g. `\"alice\"`). Used as the JWT `sub` claim.",
      type: "string",
      required: true,
    },
    partyId: {
      description: "Canton partyId (e.g. `\"Alice::122059a10c...\"`). Added to `actAs` and `readAs` claims.",
      type: "string",
      required: true,
    },
    expiresInSec: {
      description: "Token lifetime in seconds.",
      type: "number",
      default: "86400",
    },
  }}
/>

## How tokens are generated

`sandboxAuth` generates a fresh HMAC256-signed JWT on each request. The payload follows the Canton Sandbox JWT format:

```json
{
  "sub": "alice",
  "aud": "https://daml.com/ledger-api",
  "scope": "daml_ledger_api",
  "actAs": ["Alice::122059a10c..."],
  "readAs": ["Alice::122059a10c..."],
  "exp": 1735689600
}
```

Tokens are not cached by default — a new token is generated per request. For high-throughput scenarios, add a `expiresInSec` value and the `JwtManager` caches until the last 60 seconds before expiry.
