**Nexus Framework** bridges the Canton JSON Ledger API v2 with the modern React ecosystem (Next.js, TanStack Query) by eliminating the "Integration Tax" — the repetitive auth, type-mapping, and cache-management work every Canton dApp re-implements.

## Architecture

```
Your App (Next.js / React)
       │
       ▼
@nexus-framework/react        ← useContracts, useCreateContract, withLedgerAction
       │
       ▼
@nexus-framework/core         ← CantonClient, JwtManager, CommandSubmitter
       │
       ▼
Canton JSON Ledger API v2     ← http://localhost:7575
```

## Key features

| Feature | Description |
|---------|-------------|
| **Type-safe HTTP client** | Zod-validated Canton API responses, no `any` |
| **Zero-config auth** | Sandbox (HMAC256), JWT passthrough, OIDC client_credentials |
| **TanStack Query integration** | `queryOptions` factories, smart cache invalidation by template |
| **SSR prefetch** | `prefetchContracts` + `HydrationBoundary` for Next.js App Router |
| **Server Actions** | `withLedgerAction` wraps any ledger op with `LedgerActionResult` |
| **Session cookies** | AES-GCM encrypted HttpOnly cookies for Next.js SSR |
